## 4. AIで対応困難な部分

### 4.1 技術的限界

1. **アニメーションの一貫性**
   - フレーム間の連続性維持が困難
   - キャラクターの同一性保持が課題

2. **ゲームバランス調整**
   - プレイテスト無しでの最適化は不可能
   - 人間のフィードバックが必須

3. **物理シミュレーション整合性**
   - 3Dモデルのリグ設定
   - コリジョン設定の精密な調整

4. **品質管理(QA)**
   - バグの発見と修正
   - ユーザビリティテスト

### 4.2 創造的限界

1. **独創的なゲームコンセプト**
   - AIは既存パターンの組合せが得意
   - 革新的アイデアは人間の発想が必要

2. **感情的体験の設計**
   - プレイヤーの感情曲線設計
   - 意図した体験の実現確認

3. **ブランドの一貫性**
   - アートスタイルの統一
   - 世界観の整合性維持

### 4.3 法的/倫理的問題

1. **著作権**
   - 学習データに由来する類似性リスク
   - 商用利用時のライセンス確認

2. **音声の権利**
   - ボイスクローンの肖像権
   - 許諾なしの使用リスク

## 5. 推奨自動化パイプライン

```
[Phase 1: 企画] 人間主導
    │
    ├─ ゲームコンセプト定義
    ├─ ターゲット層決定
    └─ コアメカニクス設計

[Phase 2: プリプロ] AI支援
    │
    ├─ LLM → シナリオ/設定生成
    ├─ SD/DALL-E → コンセプトアート生成
    └─ 人間 → 方向性承認

[Phase 3: 制作] 高度自動化
    │
    ├─ 画像AI → 2Dアセット大量生成
    ├─ 音声AI → BGM/SE/ボイス生成
    ├─ LLM → コード/データ生成
    └─ 人間 → 品質チェック/調整

[Phase 4: テスト] 人間必須
    │
    ├─ プレイテスト
    ├─ バランス調整
    └─ バグ修正

[Phase 5: リリース] 自動化可能
    │
    ├─ ビルド自動化
    └─ デプロイパイプライン
```
