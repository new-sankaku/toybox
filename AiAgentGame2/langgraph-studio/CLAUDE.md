# プロジェクトルール

## 禁止事項

1. **応急処置の禁止** - インラインスタイル（`style={{ }}`）でのべた書きは禁止。必ずTailwindクラスまたはCSSファイルを使用すること。

2. **動作確認なしの大量変更禁止** - 複数ファイルを一度に変更する前に、1ファイルずつ動作確認すること。

3. **既存コンポーネントの破壊禁止** - Card, CardHeader等の既存UIコンポーネントを使わずに書き換えることは禁止。

4. **推測での修正禁止** - 「たぶんこれだろう」で修正を始めない。必ず問題の要素を特定してから修正すること。

## スタイル修正の正しい手順

1. まず `tailwind.config.cjs` の色定義を確認
2. 次に `src/styles/index.css` のCSS変数を確認
3. 問題があれば設定ファイルを修正
4. コンポーネントではTailwindクラスを使用

## CSS問題のデバッグ手順

1. **問題の要素を特定** - どのコンポーネントの、どの部分か明確にする
2. **コンポーネントを読む** - 該当するTSXファイルを開き、使用しているclassNameを確認
3. **CSS定義を確認** - `index.css`で該当クラスの定義を読む
4. **継承関係を追跡** - 親要素の色設定も確認する
5. **それから修正** - 原因を特定してから初めてコードを変更する
